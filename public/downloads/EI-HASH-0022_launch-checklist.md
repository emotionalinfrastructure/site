# Public Launch Checklist

Before publishing the EIT public portfolio, the artifact library must be cleaned so that each public file has a stable filename, a status label, and a clear description. The first release should include only the most polished public documents, because a smaller controlled release is more credible than a large unfiltered archive.

The homepage must be reviewed for accuracy. It should state that EIT is a candidate research, standards, policy, and implementation framework. It should not claim external adoption, certification, peer-reviewed publication, legal compliance approval, or formal standards recognition unless those facts are documented.

The research page must identify manuscript status accurately. The standards page must identify candidate status accurately. The implementation page must avoid publishing private source code, raw logs, credentials, system secrets, or unreviewed prototypes. The downloads page must separate public-ready files from restricted files. The contact page must create a professional inquiry path without promising services that have not been formally structured.

Every download link must be tested. Every file should be virus-scanned before public release. Every public PDF should open correctly. Every zip file should be inspected before release. Every document should be checked for accidental private information, personal identifiers, irrelevant records, unredacted logs, or claims that exceed the actual evidence.

After publication, the release manifest should be updated with the public URL, release date, document hashes, and version number. Any later revision should receive a new version label rather than silently replacing the original public artifact.
