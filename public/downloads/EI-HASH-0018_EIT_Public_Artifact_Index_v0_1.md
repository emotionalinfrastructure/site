Raw logs, legal templates, and unreviewed code should not be placed on the public homepage without redaction, security review, or attorney review.

## Canonical Public Order

The first public tier should present `emotional_infrastructure_full_report.pdf` as the fast executive overview, `EIT_Paper2_Updated_Submission_Package/docs/EIT_Paper2_Updated_APA7_Manuscript.pdf` as the strongest protocol-level research artifact, `EIT_Paper1_APA7_Manuscript.pdf` as the companion academic manuscript, `Wright_Trajectory_Governance_Manuscript.pdf` as the education-focused flagship manuscript, and `EI_Spec_v1_Governing_Edition.pdf` as the standards seed document. This order gives readers a controlled path from concept to research to standards to implementation.

The second public tier should present the figure and schema materials inside the Paper 2 package because they prove the work is not only expressive theory but also operational governance logic. The SDK and SaaS prototype should be presented only after code review, dependency audit, license selection, and README expansion. The contract rider should be described as attorney-review required and should not be sold as legal advice. The raw forensic log should remain private unless a separate redaction process removes personal identifiers and unrelated application references.

## Integrity and Versioning Rule

Every artifact released from this library should keep its filename, version number, creation date, and SHA-256 hash in the release record. A public portfolio may link only the polished PDFs and selected zipped packages, while the private master archive keeps editable Word documents, raw logs, and internal databases.
